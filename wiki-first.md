# wiki-first

[![Quillx](https://raw.githubusercontent.com/qainsights/Quillx/main/badges/quillx-4.svg)](https://github.com/qainsights/Quillx) Generated by Claude under human direction. Refined through production use.

Ask the local wiki before reaching for the web. A small service serves the operator's `llm-wiki` corpus air-gapped on loopback, and every citation it returns has already been checked against the git blob it names and the chunk it quotes, so a cited answer is one that needs no re-verification.

## What it does

Domain: the local `llm-wiki` corpus. Reader: whoever is about to reach for WebSearch or WebFetch and should check local coverage first.

The mechanism is a single HTTP call, a question posted to the loopback service, answered with a JSON object carrying a verdict and a set of citations. Each citation names the source document and the exact git blob it's pinned to, so a reader who wants the underlying source can go straight to it rather than trusting the summary. If the connection is refused, the service simply isn't running: the skill's answer is to fall back to normal research and note in the report that the wiki wasn't consulted, not to treat the refusal as an error worth working around.

This is the companion gate to the `research` skill: `research` is the house layer for external lookups once you're doing them, this skill is the check that runs first, before deciding an external lookup is needed at all.

## Verdicts and what to do with them

- **ANSWERED**: use it as-is. Every citation has already been checked; no web search is needed for this question.
- **PARTIAL**: use the cited part. The response's `could_not_establish` field lists exactly what the corpus doesn't support, so only that part goes to the web.
- **COULD_NOT_ESTABLISH**: the corpus can't answer this one. Search the web now, then ingest what's found so the next agent doesn't repeat the same search.

Each citation also carries a `weakly_sourced` flag: `true` means the claim rests on a tertiary source or a single secondary one, and should be treated as a lead rather than a fact, said as such in the report; `false` means it's fully corroborated.
