# interview

[![Quillx](https://raw.githubusercontent.com/qainsights/Quillx/main/badges/quillx-4.svg)](https://github.com/qainsights/Quillx) Generated by Claude under human direction. Interview methodology, question categories, and spec format were defined by a human and refined through real-world use.

A Claude skill for extracting technical requirements through structured interviews before building begins. Produces actionable spec files that builders can work from without guessing at intent.

## What it does

Conducts a multi-phase requirements interview: explores the codebase for context, researches the domain for standards and pitfalls, asks targeted questions organized by category, runs gap analysis via the requirements-auditor agent, and writes a structured spec file.

Designed for use before any non-trivial build task. The skill auto-calibrates interview depth to the request's complexity (quick/standard/thorough) with optional user override.

## Covers

- **Triage**: greenfield vs modification detection, codebase exploration, depth calibration
- **Domain research**: standards, common pitfalls, libraries via WebSearch and context7
- **Structured interview**: core functionality, scope boundaries, integration points, NFRs, edge cases, acceptance criteria
- **Gap analysis**: spawns requirements-auditor agent for completeness review, asks follow-ups for critical gaps
- **Spec production**: writes `specs/<date>-<slug>.md` with tables for functional requirements, NFRs, integration points, edge cases, assumptions, and open questions

## Interview technique

- Skips questions the codebase already answers
- References specific files and patterns discovered during exploration
- Records "I don't know" as assumptions with defaults and sensitivity notes
- Never asks googleable questions (that is the domain research phase's job)
- Adapts question count to calibrated depth

## Companion agents

- **requirements-auditor**: spawned during gap analysis to find missing requirements, unstated assumptions, and incomplete acceptance criteria
- **plan-auditor**: recommended post-interview for reviewing the implementation plan built from the spec

## Also available as
