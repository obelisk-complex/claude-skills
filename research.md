# research

[![Quillx](https://raw.githubusercontent.com/qainsights/Quillx/main/badges/quillx-4.svg)](https://github.com/qainsights/Quillx) Generated by Claude under human direction. The source order, the control-slug and content-grep verification steps, and the model-cost policy were defined by a human and refined through production use.

The house layer over generic deep research: this operator's own source order, a concrete two-step verification procedure, and a model-cost discipline that stops mechanical lookups from silently running on an oversized model. Consult before dispatching any research subagent or doing an external lookup directly.

## What it does

Domain: sourced external research, search, fetch, extract, verify, cite. Reader: whoever is about to dispatch a research subagent or do an external lookup directly, before acting on what it finds.

Every factual claim has to carry a URL that was actually fetched, not one constructed from memory or pattern-matched from a domain's URL structure. `UNVERIFIED` is treated as a praised, first-class outcome rather than a failure to hide, a gap honestly marked is more useful than a citation that dissolves under checking. This matters because agents have fabricated plausible-looking archive URLs when asked for one they hadn't actually fetched.

It also fixes a quieter problem: a subagent with no `model:` in its own frontmatter inherits the parent session's model, so on a high-end session every dispatched research task runs at that tier by default, including the purely mechanical ones. The skill assigns Sonnet to search/fetch/extract/cite and to mechanical comparison or summarisation work, Haiku to bulk fetch and dedup where throughput matters more than judgement, and reserves a larger model for genuine cross-source adjudication, weighing which of several contradictory sources is authoritative.

Findings are written incrementally: a skeleton report file goes down before the first search, each finding is appended the moment it passes verification, and status flips to done only when the research is actually complete, so a run that's interrupted at 90% still leaves 90% of the findings on disk. Fan-out dispatches carry the scope, the question, an output file path, Write access, and the sourcing gate spelled out explicitly, never "based on your findings, fix it."

## Verification procedure

An HTTP 200 alone proves nothing, many sites soft-404, serving a templated "not found" page with a 200 status. Two checks, both required, before a claim is marked verified:

1. **Control-slug test**: fetch a deliberately bogus slug on the same domain and confirm it doesn't also return 200. If it does, the domain can't be trusted on status codes at all, and verification falls back entirely to content.
2. **Content grep**: fetch the actual cited URL and grep the body for the specific fact being claimed, not just confirm it loaded. No match means the page doesn't say what's being cited, the claim is downgraded to `UNVERIFIED` or dropped, never kept as a citation anyway.

If either check can't be run in the current environment, that's said explicitly rather than reporting the claim as verified regardless.

## House sources

A local `llm-wiki` corpus is source of truth, consulted before any web search. Beyond it, web queries go through Brave Search, keyed via an environment variable read from a project `.env` file rather than anything hard-coded. Once an external lookup returns something worth keeping, it's ingested back into the wiki, raw source, summary page, index, and log, before the task counts as complete; an external fact that never makes it in gets re-fetched, or re-guessed, next time.
